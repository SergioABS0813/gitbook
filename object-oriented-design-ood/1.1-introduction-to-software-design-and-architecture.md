# 1.1 Introduction to Software Design and Architecture

### Software Design vs Architecture

| Software Design                                                                                                                                                               | Software Architecture                                                                                                                                                      |
| ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Analiza un componente del sistema (Bajo nivel)                                                                                                                                | Analiza el sistema completo (Alto nivel)                                                                                                                                   |
| <p>Responde a las preguntas: ¿Qué parámetros necesita este método?<br>¿Qué patrón de diseño resuelve mejor este problema en el código?<br>¿Cómo se estructura esta clase?</p> | <p>Responde a la pregunta:<br>¿Qué arquitectura tendrá el sistema?<br>¿Qué tecnologías usaremos (frameworks, soluciones)?<br>¿Cómo se comunicarán los componentes?<br></p> |
| Traduce necesidades del cliente en código funcional en cada componente                                                                                                        | Traduce las necesidades del negocio/cliente en una visión global del sistema.                                                                                              |

{% hint style="info" %}
#### Ejemplo práctico

"Si el arquitecto de software decide usar MySQL como base de datos del sistema, el diseñador de software modela las entidades y las relaciones entre cada una de ellas y qué patrones de diseño se usarán para que el código esté más limpio."
