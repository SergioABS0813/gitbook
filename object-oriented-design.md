# Object Oriented Design

Software Desgin: Analiza el sistema a bajo nivel, o sea define detalladamente los componentes (¿Qué parámetros necesita este método?, ¿Qué patrón de diseño resuelve mejor este problema de código?, ¿Cómo se estructura esta clase?). Traduce las necesidades del cliente en código que sea opere correctamente.

Software Architecture: Analiza el sistema a alto nivel, o sea decide sobre la arquitectura de todo el sistema y las tecnologías que usaremos (frameworks, soluciones y cómo los componentes interactúan).
